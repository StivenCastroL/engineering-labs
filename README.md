# Engineering Labs

Laboratorio personal de experimentación técnica en networking, infraestructura, seguridad e inteligencia artificial.

Cada laboratorio sigue una metodología estructurada de documentación que incluye: objetivo, hipótesis, arquitectura, procedimiento, validaciones, resultados, problemas encontrados, lecciones aprendidas y mejores prácticas.

## Áreas de conocimiento

| Área | Descripción |
|------|-------------|
| [Networking](docs/networking/index.md) | Cisco SD-WAN, Routing, Switching, Fortinet |
| [Infrastructure](docs/infrastructure/index.md) | Proxmox, virtualización, containers, storage |
| [Security](docs/security/index.md) | SIEM, EDR, Zero Trust, hardening |
| [AI](docs/ai/index.md) | AI Agents, Local LLMs, automatización |

## Metodología

Todos los laboratorios se documentan siguiendo una estructura estándar:

1. **Información general** — tecnología, nivel, entorno, fecha
2. **Objetivo** — qué se busca aprender o validar
3. **Contexto técnico** — fundamentos de la tecnología
4. **Arquitectura** — topología, nodos, direccionamiento
5. **Hipótesis** — comportamiento esperado a validar
6. **Procedimiento** — paso a paso reproducible
7. **Validaciones técnicas** — comandos y outputs reales
8. **Resultados** — qué ocurrió durante el lab
9. **Problemas encontrados** — errores, causas y soluciones
10. **Lecciones aprendidas** — conocimiento derivado del experimento
11. **Mejores prácticas** — recomendaciones de ingeniería
12. **Próximos experimentos** — cadena de aprendizaje progresiva
